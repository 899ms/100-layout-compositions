# Release Checklist

Use this checklist when publishing the project on GitHub.

1. Create the repository on GitHub.
2. Replace `your-name/100-layout-compositions` in `README.md` with the real
   GitHub owner and repository name.
3. Push the repository.
4. Create a GitHub Release, for example `v1.0.0`.
5. Upload `dist/100-layout-compositions-images.zip` as a release asset.
6. Confirm that the README download link works.

The ZIP is intentionally ignored by Git because large binary archives should not
be committed to repository history.
